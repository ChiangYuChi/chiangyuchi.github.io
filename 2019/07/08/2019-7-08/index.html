克服 JavaScript 的奇怪部份 Day29-物件、函數與 this

Jul 8, 2019

這一節要繼續深入了解到底什麼是「函數」？ 函數是一種物件，具有屬性及其他特徵。現在要繼續探究的是，回到執行環境來看，在函數被呼叫的時候，底層究竟發生什麼變化呢？ 現在要繼續深究物件、函數以及有時讓人感到困惑的關鍵字 ”this” 之間的關係。

必須放在心中的是，當函數被呼叫的時候，會創造新的執行環境，但是此處不要和上面提到的物件搞混了。記憶體裡的物件，也就是有屬性與方法的函數。它有名稱屬性和程式碼的屬性，是程式碼存在的地方。當程式被呼叫的時候，執行環境被創造，放進執行堆。這決定了程式的執行方式。如果想一想執行環境，關注在函數物件的程式部分，當我執行程式屬性裡面的程式會怎麼樣？

我們知道執行環境被創造的時候，每個執行環境有自己的變數環境，也就是創建在函數裡面的變數所在的地方，它可以參照到外部詞彙環境，也就是它在程式碼當中的實際位置，讓它能夠隨著範圍鏈一路往下找。意思是如果我取用一個變數，而不在函數的變數環境中，它匯到外面找，尋找這個變數或是函數，直到全域環境為止，它有一個外部參照。

但是每當 在 JavaScript 引擎裡，每一次有一個執行環境創造的時候，每當函數被執行，JavaScript 引擎會給我們一個不曾宣告的東西。它會給我們一個很有用的 this 變數。this 會根據函數被呼叫的方式，而指向不同的物件、不同的東西。這會造成許多混淆。在少數情況下，this 會依據函數如何被呼叫而改變。JavaScript 引擎會決定this應該指向別的東西。

我們已經看過 this 是可以取用的，即使在全域執行環境也一樣。

在瀏覽器內執行，它會顯示 window 物件。因為 this 會指向全域物件，在瀏覽器內的全域物件就是 window 物件。

現在看另外一個例子，有一個命名為 a 的函數，呼叫 a 就等於執行程式屬性，也就是函數裡面全部的程式。創造執行環境時，其中一部分就是創造 this 關鍵字。那在呼叫 a 的時候，this 關鍵字會在執行環境裡面變成什麼呢？仍然是一個 window 物件。所以創造一個函數的時候，this 關鍵字仍然會指向全域物件。

如果我使用函數表示式設定物件，然後再將它設定為一個變數，變數 b 用函數表示式去創造、宣告。 在這個例子中的 this 會變成什麼呢？

呼叫 b 的結果，this 仍然指向 window 物件。

所以無論何時我創造函數，用函數表示式或函數陳述句，在此處創造函數，this 會指向全域物件。即使有 3 個執行環境，在每一種情況中，它們都有自己的 this 關鍵字，但在所有的例子當中，this 都指向記憶體中的同一個位址，也就是全域物件。

根據 this 特性做的特殊操作

根據這種特性，我們可以使用例如 this.newvariable = 'hello'; ，將它連結到全域物件。所以在呼叫 a 之後，我們可以用 console.log 查詢任何連結到全域物件的變數，都可以這樣參照到它，而不需要用點運算子。它會假定我要取用全域物件的變數

重新整理之後，會發現印出 hello。

如果不了解 this 會指向哪裡，或是覺得他會連結到函數，就可能會和全域命名空間產生衝突，導致很多問題。

物件方法

那麼物件方法呢？現在我們已經了解函數表示式，我們可以做一些其他可以在物件實體裡面做的事情，我們可以創造方法。

如果純值稱為屬性，但如果值是一個函數，則稱為方法。現在建立一個名稱/值配對，現在呼叫被創造在物件實體的函數，那 this 會是什麼？

當函數被呼叫的時候，新的執行環境就被創造，JavaScript 引擎決定 this 關鍵字要指向誰。在其他例子中，會指向 window 物件，但在這個例子，它會指向物件的方法。

常見誤解

假設我們在方法裡面創造一個函數，現在我們預期會在第二個 log 裡面看到 update again the c object ，因為 this 仍然會指向物件，而我可以改變它。

這是為什麼？因為這裡的名稱屬性被等號運算子創造新增到全域物件。這表示當這個裡面的環境，它的執行環境被創造時，this 指向全域物件。即使它處在我所創造的物件裡面。

這會被認為是一種 bug，但這個情況下 JavaScript 就是如此運作的。當我們遇到這種情形時能怎麼辦呢？

常用解決模式

既然我們已經知道物件是用 by reference 設定的，我們可以設定一個變數將它設定為 this。這是物件方法中的第一行。我們有一個新的變數，而且物件都是 by reference，所以 self 會指向和 this 一樣的記憶體位置，現在第一行的 this 會指向整個物件。在一般使用this 的時候，都改用 self，即使在子函數裡面。這樣我就不需要考慮是否指向對的物件。

由於 self 沒有在函數裡面被宣告，所以 JavaScript 引擎會往範圍鏈裡面找。這個函數的實際位置在哪裡？所以我們可以接著往外一層看，所以到外部環境找到一個叫做 self 的變數。所以在這個函數裡的self，會成為第一行的那一個。而且仍然指向整個物件。
